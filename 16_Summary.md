### 프로세스 마이닝이 제공하는 4가지 가치 — 이 교육을 통해 경험

이 교육은 프로세스 마이닝이 업무 현장에서 제공하는 네 가지 가치를 **Jira 티켓팅 3,192건의 실제 데이터로 검증**한 과정이었습니다.

- **구조적 문제의 자동 정량화** — 이벤트 로그에서 "이탈률 70% · 리드타임의 74%가 2개 Change 활동에 집중" 같은 구조적 병목을 수동 인터뷰 없이 추출
- **손실의 금액 단위 측정** — 참조 모델(reference model) 대비 이탈을 € · 시간으로 환산 → "이탈 시 평균 +$14(≈€13)/티켓 · +7일 리드타임 · 인건비 10%(€9,770) 예방 가능 낭비"
- **조건부 ROI 예측** — 실제 로그 패턴 기반 What-if로 가정이 아닌 조건부 예측 → "건당 €2.99 · 연간 €19,000 절감 · 투자 €15,000 첫해 회수"
- **분석 결과의 현장 자동 연결** — 수동 모니터링 없이 위반이 사람을 호출 → "실행 중 47개 티켓 감시 뷰 + Slack 자동 알림"

### 무엇을 했나 — 5단계 여정

각 실습은 독립 수행 방식이 아니라, 단계별 결과물을 다음 단계에 순차적으로 반영하는 형태로 설계되어 있습니다.

| 실습 | 핵심 작업 | 주요 산출물 |
|------|----------|-------------|
| **1. 데이터·프로세스 구성** | 원시 CSV → 이벤트 로그 + 비용 모델 + BPMN 참조 모델 | 분석 재료 |
| **2. 발견·초기 분석** | 4개 모델 뷰 + 적합성 검사 + 처방적 AI 보고서 | 정량 진단(이탈률 70% · 범인 2개 활동) |
| **3. 대시보드·상황별 분석** | SLA 사용자 정의 지표 + 5종 대시보드 + 객체 테이블 | 현장 운영 도구 |
| **4. What-if 시뮬레이션** | 근본 원인 제거 시나리오 + historical 기반 ROI 계산 | 사업성 검토 근거(€19K/년 · 첫해 회수) |
| **5. 운영 우수성 모니터링** | 레거시 복구 + 실행 중 감시 뷰 + Slack 자동 호출 | 실시간 안전망 |

**흐름** : 데이터 → 진단 → 운영화 → 예측 → 현장 호출

### 주요 산출물(Asset) — 3가지 범주

| 범주 | 산출물 | 실무 쓰임 |
|------|--------|----------|
| **분석 재료** | 이벤트 로그 · 비용 모델 · BPMN 참조 모델 | 모든 정량 분석의 기반 |
| **운영 도구** | 5종 대시보드 · 사용자 정의 지표 · 객체 테이블 · 모니터+서비스 | 현장 의사결정 + 실시간 감시 |
| **사업성 검토 근거** | What-if 시뮬레이션 프레임 · ROI 비즈니스 케이스 | 자동화 투자 승인 |

Jira 티켓팅을 넘어 구매 · 송장 · 보험 청구 · 온보딩 등 **단계가 있는 모든 업무 프로세스**에 동일 적용 가능합니다.

### 그래서 왜 IBM Process Mining인가 — 4가지 차별점

위 4가지 가치는 여러 프로세스 마이닝 도구가 공통으로 제공합니다. 이 교육에서 확인된 **IBM Process Mining만의 차별점**은 다음 네 가지입니다.

<strong>1. 분석을 실행으로 연결하는 "처방적 AI" — Insight to Action</strong>    
분석가가 수일 걸릴 근본 원인 분석을 **1분 내 자동 수행**하고, 결론에 더해 **담당자 재배정 권고 + 이메일 초안**까지 함께 제시합니다. 다른 도구가 "무엇이 문제인가"에서 멈춘다면, IPM은 "지금 누구에게 무엇을 지시할 것인가"까지 이어집니다. *(실습 2)*

<strong>2. 데이터 수정 없이 확장되는 "유연한 분석 구조" — 데이터 준비 비용 최소화</strong>    
**사용자 정의 지표**가 이벤트 로그에 없는 비즈니스 규칙(SLA 등)을 주입하고, **객체 테이블**이 외부 마스터 데이터(국가 · 벤더 · 자재)를 분석 시점에 결합합니다. 데이터 파이프라인을 고치지 않고도 분석 범위가 수평 확장됩니다. *(실습 3 · IPM 2.0 신기능)*

<strong>3. 가정이 아닌 실제 데이터 기반 "신뢰 가능한 시뮬레이션" — 의사결정에 바로 쓰는 결과</strong>    
What-if의 4대 입력(도착 분포 · 직원 가용성 · 활동 통계 · 대기 시간)이 **실제 로그에서 자동 추출**됩니다. 분석가가 "대충 이럴 것"을 입력하는 스프레드시트 예측과 달리, <strong>"과거 패턴의 연장선에서 특정 변수만 바뀐 미래"</strong>를 계산합니다. *(실습 4)*

<strong>4. 분석에서 실행까지 이어지는 "End-to-End 운영 플랫폼"</strong>    
**Monitor**가 Analytics 테이블 위젯의 항목 존재를 트리거하고, **Service**가 이메일·Slack·IBM AppConnect로 자동 발송합니다. 대시보드를 수동 주시하는 체계가 **위반이 사람을 호출하는 자동 체계**로 전환되며, IBM Cloud Pak에 속해 있어 Business Automation Workflow · Decision · RPA와 통합 운영도 가능합니다. *(실습 5)*

### 실전 적용 — 여러분 조직의 다음 프로세스에

1. 이벤트 로그를 **비용으로 환산 가능한 상태**로 만든다 *(실습 1 패턴)*
2. **4-렌즈 + 참조 모델 + 처방적 보고서**로 범인 활동을 확정한다 *(실습 2 패턴)*
3. SLA · 비즈니스 규칙은 **사용자 정의 지표로 주입**한다 *(실습 3 패턴)*
4. **근본 원인 제거안**을 골라 historical 시뮬레이션으로 ROI를 산출한다 *(실습 4 패턴)*
5. **자동화 완성을 기다리지 말고 운영 모니터부터 켠다** *(실습 5 패턴)*
